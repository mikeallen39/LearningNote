**Ling-V2** 是由 **InclusionAI**开源的基于 **MoE (Mixture of Experts)** 架构的大语言模型。

Ling-V2 的核心目标是**在极低的激活参数量下实现媲美更大规模稠密模型（Dense Model）的性能，同时提供极高的训练和推理效率**。

以下是对 Ling-V2 的详细解读：

### 1. 核心模型：Ling-mini-2.0
目前开源的重点模型是 **Ling-mini-2.0**，它展示了“小激活、大总参”的 MoE 优势：
*   **参数规模**：总参数量 **16B**，但每次前向传播仅激活 **1.4B** 参数（非嵌入层仅 789M）。
*   **性能对标**：尽管激活参数很少，其性能却能达到 **7B-8B 稠密模型** 的水平（官方称具有 **7倍** 的等效稠密性能杠杆率）。
*   **训练数据**：基于超过 **20T (20万亿)** 高质量 Token 训练，并经过多阶段 SFT 和 RL 优化。
*   **能力表现**：在代码（LiveCodeBench, CodeForces）、数学（AIME 2025, HMMT 2025）以及专业知识推理（MMLU-Pro）上表现优异，甚至超越了一些更大的 MoE 模型。

### 2. 关键技术创新
Ling-V2 在架构和工程实现上有几个显著亮点：

*   **极致稀疏架构 (1/32 Activation Ratio)**：
    *   采用了 **1/32** 的激活比例，即每 32 个专家中只激活 1 个。
    *   引入了优化的专家粒度、共享专家比例、Attention 比例等设计。
    *   使用了 **Aux-loss free + Sigmoid routing** 策略，无需辅助损失即可平衡专家负载。
*   **FP8 高效训练方案**：
    *   全流程支持 **FP8 混合精度训练**。实验表明，相比 BF16，FP8 在损失曲线和下游任务表现上几乎无损。
    *   开源了包括 FP8 Optimizer、按需转置权重、填充路由图等在内的完整 FP8 训练解决方案，极大降低了显存需求。
*   **推理速度极快**：
    *   在简单问答场景下（2000 tokens 内），在 H20 显卡上生成速度可达 **300+ tokens/s**，比 8B 稠密模型快 **2倍** 以上。
    *   随着上下文长度增加（支持 **128K** 上下文，通过 YaRN 扩展），相对加速比可达 **7倍**。
*   **MTP (Multi-Token Prediction) 支持**：
    *   架构支持多 token 预测技术，进一步提升了训练吞吐和推理速度。

### 3. 开源策略与模型版本
InclusionAI 采取了非常开放的策略，不仅发布了后训练模型，还释放了完整的预训练检查点，便于社区研究：

*   **Ling-mini 系列 (1.4B 激活 / 16B 总参)**：
    *   **基座模型 checkpoints**：分别开放了训练至 **5T, 10T, 15T, 20T** Token 的基座版本，以及最终的 `Ling-mini-base-2.0`。
    *   **对话模型**：`Ling-mini-2.0` (支持 32K -> 128K 上下文)。
*   **Ling-flash 系列 (6.1B 激活 / 100B 总参)**：
    *   提供了更大规模的版本 `Ling-flash-2.0` 及其基座，适用于更复杂的任务。
*   **格式支持**：支持 Safetensors 格式，并提供工具将训练产生的 DCP 格式转换为 Safetensors (支持 BF16 和 FP8)。

### 4. 部署与生态支持
*   **推理框架**：
    *   **Hugging Face Transformers**：原生支持，代码示例简单。
    *   **vLLM**：支持离线批处理和在线 API 服务。需注意目前需要应用官方仓库提供的 `patch` 才能完美支持其 MoE 结构和 MTP。支持 YaRN 长上下文扩展。
    *   **SGLang**：同样需要应用 patch，支持 BF16 和 FP8 模型推理，支持 Speculative Decoding (MTP)。
*   **训练框架**：
    *   提供了完整的预训练和微调代码。
    *   推荐使用 **Llama-Factory** 进行微调，也支持 Megatron。
    *   **NPU 支持**：特别提到了对华为昇腾 (NPU) 的支持，提供了在 NPU 上进行继续预训练、微调和推理的 Demo。

### 5. 性能数据对比 (训练吞吐)
在 8x/16x/32x 80G GPU 环境下，Ling-mini-2.0 的训练吞吐量显著优于同级别的稠密模型：
*   相比 LLaMA 3.1 8B，吞吐量提升 **30%-40%** (开启 MTP)。
*   若关闭 MTP，吞吐量提升甚至可达 **90%-120%**。
*   相比 Qwen3 8B，优势更为明显（Qwen3 在此测试中表现低于 LLaMA 3.1 基线）。

### 总结
**Ling-V2 (特别是 Ling-mini-2.0)** 是当前 MoE 领域的一个标杆性开源项目。它证明了通过极致的稀疏化设计（1/32 激活率）和工程优化（FP8 训练、MTP），可以在**极低的推理成本**（1.4B 激活参数）下获得**主流 7B-8B 模型的性能**。
